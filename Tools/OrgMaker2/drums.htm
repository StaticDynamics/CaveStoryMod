See section Cave Story.

See section Other OrgMaker versions.

Cave Story

I'm using OrgMaker 2.1.0x English to compose an ORG file that I will insert into a Cave Story mod. What drums should I use?

Use these drums

  • Track Q: Bass01
  • Track W: Snare01
  • Track E: HiClose
  • Track R: HiOpen
  • Track T: Tom01
  • Track Y: Per01
  • Track U: Bass01 (and Track U must remain empty)
  • Track I: Bass01 (and Track I must remain empty)

In the Org210x.exe release dated 2016-03-04, when you make a new file, it will use these drums by default.

More information

Cave Story only uses the drum tracks Q to Y. Cave Story ignores the drum specified in the ORG file and always uses the hard-coded drum sounds above. By using the drums above when you compose, your ORG file will sound the same in OrgMaker 2.1.0x English as it will in Cave Story.

Tracks U and I aren't used by Cave Story and must remain empty. If these tracks contain any notes, Cave Story will crash.

If any drum track uses a drum that's incompatible with OrgMaker 1.3.4 (see the next section), the file header will change, and Cave Story won't play the ORG file at all. In OrgMaker 2.1.0 Japanese and in OrgMaker 2.1.0x English releases dated 2014-09-20 or before, when you start a blank file, every drum track defaults to a drum that's incompatible with OrgMaker 1.3.4. It's important to change Track U and I to an OrgMaker 1.3.4-compatible drum like Bass01 even though you have to leave these tracks empty. Otherwise Cave Story won't play the ORG file at all.

Other OrgMaker versions

When I compose in OrgMaker 2.1.0x English, will the drums work in other versions of OrgMaker?

  • Green: Will sound the same or very close to the same.
  • Yellow: Will sound different. (Swap fix available*)
  • Orange: Will sound different. (No fix available)
  • Red: OrgMaker won't load the file, or the file will cause OrgMaker to crash.

* If it is important to you that it sound the same, when you are done composing in OrgMaker 2.1.0x English, make a copy of the file, and in the copied file make the change indicated in the table. Now the copied file played in the indicated OrgMaker version will sound the same as the original file played in OrgMaker 2.1.0x English.

More information

In Rxo Inverse's Japanese versions of OrgMaker 2, the Cave Story drums (Bass01, Snare01, HiClose, HiOpen, Tom01, and Per01) sound different than they do in Pixel's OrgMaker 1.3.4. Rxo Inverse moved the original sounds to new names (Bass04, Snare03, HiClose02, HiOpen02, and Tom03), except for Per01. (The original Per01 sound is not available in OrgMaker 2.0.5.)

In the English translations of OrgMaker 2 by Xaser and Bavi_H, the original OrgMaker 1.3.4 drum sounds have been restored to their original names, and the new sounds moved to the new names.

In OrgMaker 2.0.5x English, Per01 was still different from OrgMaker 1.3.4.

In OrgMaker 2.1.0x English, Per01 has now been copied from OrgMaker 1.3.4, so it sounds the same again.
